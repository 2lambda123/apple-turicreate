81 /**
82  * \ingroup fileio
83  * \internal
84  * Lists objects or prefixes prefixed by a give s3 url.
85  *
86  * This is a thin wrapper around the S3 API
87  * http://docs.aws.amazon.com/AmazonS3/latest/API/RESTBucketGET.html
88  * and may not quite do what you think it does.
89  *
90  * if s3_url points to a valid prefix, it will return only the prefix
91  * as a directory. For instance if I have an S3 bucket containing
92  *
93  * foo/hello.txt
94  *
95  * list_objects("s3://foo") will return simply "foo/" as a directory.
96  *
97  * See list_directory() and is_directory() for a more sensible implementation
98  * which behaves somewhat more file system like.
99  *
100  * \returns A list_objects_response object.
101  * If list_objects_response.error is an empty string, it indicates success.
102  * Otherwise, it contains an error code. list_objects_response.directories
103  * indicate all "directories" stored with the requested prefix. And
104  * list_objects_response.objects indicates all regular objects stored with the
105  * requested prefix.
106  *
107  */
108 list_objects_response list_objects(std::string s3_url,
109  std::string proxy = "");

114  * \internal

119  *
120  * foo/hello.txt

126  * \returns A list_objects_response object.
127  * If list_objects_response.error is an empty string, it indicates success.
128  * Otherwise, it contains an error code. list_objects_response.directories

130  * list_objects_response.objects indicates all regular objects stored with the
131  * requested prefix.

133  */
