39 /**
40  * Provides a query interface to a collection of statistics about an SArray
41  * accumulated via various sketching methods.
42  * The unity_sketch object contains a summary of a single SArray (a column of
43  * an SFrame). It contains sketched statistics about the Array which can be
44  * queried efficiently.
45  *
46  * The sketch computation is fast and has complexity approximately linear
47  * in the length of the Array. After which, all queryable functions in the
48  * sketch can be performed nearly instantly.
49  *
50  * The sketch's contents vary depending on whether it is a numeric array,
51  * or a non-numeric (string) array, or list type list vector/dict/recursive
52  * If numeric:
53 
54  * This is essentially a union among a collection of sketches, depend on value
55  * type of SArray, here is what's availble in the sketch for each sarray value type:
56  * numeric type (int, float):
57  * - m_numeric_sketch -- numeric summary like max/min/var/std/mean/quantile
58  * - m_discrete_sketch -- discrete summary like unique items/counts/frequent_items
59  * string type:
60  * - m_discrete_sketch -- discrete summary like unique items/counts/frequent_items
61  * dictionary type:
62  * - m_discrete_sketch -- discrete summary like unique items/counts/frequent_items
63  * - m_dict_key_sketch -- sketch summary of flattened dict keys, it is a
64  * sketch summary of string type where we treat each key as string
65  * - m_dict_value_sketch -- sketch summary of flattened dictionary values.
66  * we infer the type of dictionary value by peek into first 100 rows of
67  * of data and then decide whether or not to use numeric sketch.
68  * - m_element_sub_sketch -- optional. Available only if user explicitly asks for
69  * sketch summary for subset of dictionary keys. The sub sketch type is
70  * of the same type as m_dict_value_sketch
71  * vector(array) type:
72  * - m_discrete_sketch -- discrete summary like unique items/counts/frequent_items
73  * - m_element_sketch -- sketch summary for all values of the vector as if the
74  * values are flattened, and the element sketch is of type float (numeric sketch)
75  * - m_element_sub_sketch -- optional. If user asks for sketch summary for certain
76  * columns in the vector value, then this will be available. It is a collection
77  * of sketches for the corresponding columns and the sketch type is numeric.
78  * list(recursive) type:
79  * - m_discrete_sketch -- discrete summary like unique items/counts/frequent_items
80  * - m_element_sketch -- sketch summary for all values of the vector as if the
81  * values are flattened. The element sketch is of type string. We convert all
82  * list values to string and then do a sketch on it
83  *
84  * The following information is provided exactly:
85  * - length (\ref size())
86  * - # Missing Values (\ref num_undefined())
87  * - Min (\ref min())
88  * - Max (\ref max())
89  * - Mean (\ref mean())
90  * - Variance (\ref var())
91  *
92  * And the following information is provided approximately:
93  * - # Unique Values (\ref num_unique())
94  * - Quantiles (\ref get_quantile())
95  * - Frequent Items (\ref frequent_items())
96  * - Frequency Count of any Value (\ref frequency_count())
97 
98  * For SArray of type recursive/dict/array, additional sketch information is available:
99  * - element_length_summary() -- sketch summary of element length.
100 
101  * For SArray of type list, there is a sketch summary for all values inside the
102  * list element. Sketch summary flattens all list values and do a sketch summery
103  * over flattened values. Each value in list is casted to string for sketch summary.
104  * The summary can be retrieved by calling:
105  * - element_summary() -- sketch summary of all list elements
106 
107  * For SArray of type array(vector), there is a sketch summary for all values inside
108  * vector element. Sketch summary flattens all vector values and do a sketch summery
109  * over flattened values. The summary can be retrieved by calling:
110  * - element_summary() -- sketch summary of all vector elements
111 
112  * For SArray of type dict, additional sketch summary over the keys and values are
113  * provided. They can be retrieved by calling:
114  * - dict_key_summary() -- sketch summary of all keys in the dictionary
115  * - dict_value_summary() -- sketch summary of all values in the dictionary
116 
117  * For SArray of type dict, user can also pass in a list of dictionary keys to
118  * sketch_summary function, this would cause one sub sketch for each of the key.
119  * For example:
120  * >>> sketch = sa.sketch_summary(sub_sketch_keys=["a", "b"])
121  * Then the sub summary may be retrieved by:
122  * >>> sketch.element_sub_sketch()
123  * Or:
124  * >>> sketch.element_sub_sketch(["key1", "key2"])
125  * for subset of keys
126  *
127  * Similarly, for SArray of type vector(array), user can also pass in a list of
128  * integers which is the index into the vector to get sub sketch
129  * For example:
130  * >>> sketch = sa.sketch_summary(sub_sketch_keys=[1,3,5])
131  * Then the sub summary may be retrieved by:
132  * >>> sketch.element_sub_sketch()
133  * Or:
134  * >>> sketch.element_sub_sketch([1,3])
135  * for subset of keys
136  *
137  **/
